## 2. 架构总览

### 2.1 实际实现架构

```
┌──────────────────────────────────────────────────────────┐
│                   server/index.ts (Express)                │
│                                                            │
│  ┌────────────────────┐    ┌──────────────────────────┐   │
│  │   REST API 层       │    │   内嵌 Web 前端           │   │
│  │                     │    │                          │   │
│  │  GET  /api/sources  │    │  HTML + CSS + JavaScript │   │
│  │  PUT  /api/sources  │    │  全部内嵌在同一个文件      │   │
│  │  POST /api/run      │    │  启动后访问 localhost     │   │
│  │  GET  /api/config   │    │                          │   │
│  │  ...                │    │  无需构建、无需打包        │   │
│  └────────┬───────────┘    └──────────────────────────┘   │
│           │                                               │
└───────────┼───────────────────────────────────────────────┘
            │
            ▼
┌──────────────────────────────────────────────────────────┐
│                    后端管道 (不变)                          │
│  sources.config.json                                      │
│       → npm run daily                                     │
│       → daily_reports/<date>/                             │
│           ├── <date>.html                                 │
│           ├── <date>.json                                 │
│           └── <date>-articles.json                        │
└──────────────────────────────────────────────────────────┘
```

**与原始规划的区别**：

| 原始规划 | 实际实现 |
|---|---|
| React 18 + Vite + Tailwind 独立 SPA | 无框架，Express 内嵌 HTML/CSS/JS |
| 静态部署到 GitHub Pages | Express 服务器，访问 `http://localhost:3456` |
| 前端读取静态 JSON 文件 | 前端通过 REST API 与后端交互 |
| 构建需要 `npm run build` | 零构建，`npx tsx server/index.ts` 直接运行 |

**选型理由**：项目本身是本地工具，不需要复杂的 SPA 框架。内嵌前端方案减少了依赖和构建步骤，启动即用，修改即时生效。

### 2.2 启动方式

```bash
npm run serve
# 访问 http://localhost:3456
```

### 2.3 端口

默认 3456，可通过 `PORT` 环境变量修改。

## 3. 已实现功能

### 3.1 仪表盘

打开页面默认进入仪表盘，展示：
- 最新报告的日期、大小、状态
- 累计报告天数
- 已启用数据源数量
- 日历控件，有报告的日期高亮标记，点击直接跳转

### 3.2 配置管理

原来需要编辑 `.env` 文件。现在可以在页面上保存配置，立即生效：

| 配置项 | 说明 |
|---|---|
| LLM 后端 | claude-cli / Anthropic / OpenAI / DeepSeek / MiniMax |
| API Key | 根据所选后端自动切换对应环境变量 |
| 报告语言 | 中文 / 英文 |
| 报告时区 | IANA 时区名 |

### 3.3 数据源管理

原来需要编辑 `sources.config.json`。现在可以在页面上：

| 操作 | 说明 |
|---|---|
| 查看列表 | 分类、名称、ID、类型、URL、语言 |
| 启用/禁用 | 开关即时生效 |
| 编辑 | 修改名称、URL、类型、分类等所有字段 |
| 添加 | 填写 ID、名称、URL 等创建新数据源 |

### 3.4 一键执行

| 功能 | 说明 |
|---|---|
| 执行日报 | `npm run daily`，实时滚动输出日志 |
| 干运行 | `npm run dry-run`，快速验证配置 |
| 进度条 | 按阶段展示进度（抓取→增强→分析→生成→完成） |
| 里程碑标记 | 当前阶段和已完成阶段以不同颜色标识 |
| 运行简报 | 结束后自动汇总成功/失败的数据源 |
| 终止任务 | 随时停止正在运行的任务 |

### 3.5 日历与历史回溯

- 月视图日历
- 有报告的日期高亮，点击直接跳转到该日报 HTML
- 支持前后月份切换

### 3.6 运行日志

在 Web 界面内查看 `logs/` 目录下的运行日志文件。

## 4. 技术栈

| 层 | 选型 |
|---|---|
| 服务端框架 | Express 5 |
| 运行时 | tsx（直接运行 TypeScript） |
| 前端 | 内嵌 HTML + CSS + JavaScript（无框架） |
| API 通信 | `fetch` + REST JSON |
| 任务执行 | `child_process.spawn` 子进程 |
| 实时输出 | 轮询 `/api/run/output` |

## 6. 数据结构

### REST API 端点

| 端点 | 方法 | 说明 |
|---|---|---|
| `/api/config` | GET/PUT | 读写 .env.local 配置 |
| `/api/sources` | GET | 数据源列表 |
| `/api/sources/:id` | PUT | 更新单个数据源 |
| `/api/sources` | POST | 添加新数据源 |
| `/api/reports/summary` | GET | 报告汇总（总数、最新、每日列表） |
| `/api/reports/list` | GET | 报告列表 |
| `/api/run/:command` | POST | 启动任务（daily/dry-run） |
| `/api/run/status` | GET | 当前任务状态 |
| `/api/run/output` | GET | 任务实时输出（轮询） |
| `/api/run` | DELETE | 终止任务 |
| `/api/logs` | GET | 运行日志列表 |
| `/api/logs/:file` | GET | 单个日志内容 |
| `/api/shutdown` | POST | 停止服务器 |

### 报告 JSON 结构（来自后端 `{date}.json`）

不变，延续后端原有的格式。前端直接消费该数据结构渲染。
