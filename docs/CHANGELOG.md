# 新增功能总览

在原项目基础上，主要新增了一个完整的 Web 管理前端（基于 Express 嵌入），实现了网页简化配置、一键运行、按日历读取日报以及若干优化，更易新手使用。

---

## 一、Web 管理前端

原项目是一个命令行工具，通过 `npm run daily` 在终端执行。现在新增了一个带有图形界面的 Web 前端，通过 `npm run serve` 启动后访问 `http://localhost:3456` 即可使用，无需操作终端。

### 仪表盘

打开页面默认进入仪表盘，展示：
- 最新报告的日期、大小、状态
- 累计报告天数
- 已启用数据源数量
- 日历控件，可查看过去每一天是否有报告，点击日期直接跳转到对应报告

### 配置管理

原来需要通过编辑 `.env` 文件来修改配置。现在可以直接在页面上：
- 选择 LLM 后端（Claude CLI / Anthropic / OpenAI / DeepSeek / MiniMax）
- 填写对应的 API Key
- 设置报告语言（中文 / 英文）
- 设置报告时区
- 所有配置保存后立即生效

### 数据源管理

原来需要编辑 `sources.config.json` 来增删数据源。现在可以在页面上：
- 查看所有数据源列表（分类、名称、类型、URL、语言）
- 开关启用/禁用
- 编辑已有数据源（名称、URL、类型、分类等所有字段）
- 添加新数据源
- 实时保存到配置文件

### 一键执行

点击"执行每日日报"按钮即可运行完整日报流程，无需在终端敲命令：
- 实时显示运行日志（滚动输出）
- 进度条显示当前阶段（抓取 → 增强 → 分析 → 生成 → 完成）
- 每个阶段有对应的里程碑标记
- 运行结束后自动显示执行简报（哪些源成功/失败）
- 可随时点击"终止"停止正在运行的任务
- 支持干运行（dry-run），只抓取不调 AI，快速验证配置

### 运行日志

Web 界面内可以查看历史运行日志，无需去 `logs/` 目录翻文件。

## 二、日历功能

新增日历面板，以月视图展示日报生成情况：
- 有报告的日期高亮标记
- 点击日期直接跳转到对应日报（HTML 页面）
- 支持前后月份切换浏览
- 方便回溯历史日报

---

## 三、同一天重复运行时的合并机制

原项目每次运行会生成带时间戳后缀的文件，同一日期可能有多份文件。优化后：

1. **每日只存一份**：移除时间戳后缀，始终写入 `{date}.json`、`{date}.html`、`{date}-articles.json`
2. **合并更新**：如果当天已经运行过，再次运行时会：
   - 加载已有的文章数据
   - 和新抓取的文章按 URL 去重合并
   - 同 URL 的保留已有 AI 摘要，新 URL 的补充进来
   - 用全部文章重新生成日报
3. 这样即使某些数据源临时抓取失败，之前的数据也不会丢失

## 四、其他优化

- **进度条更准确**：修复了日志中 "enrichment done" 等中间状态误触发完成标记的问题；调整了各阶段进度占比，更贴近真实耗时
- **任务完成检测更稳定**：统一了服务器返回格式，前端轮询更可靠
- **停止任务时正确收起进度条**：修复了终止后界面残留的问题
- **运行确认提示**：当天已有报告时弹出确认，避免误覆盖
